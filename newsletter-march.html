New! Error Categorization

We've created three new error classes. These will affect how errors are grouped and how notifications are sent. They are managed on a per project level.

  • Global error classes
    • Any error with this categorization will be grouped with other errors of its class, regardless of where they happen in your application.

  • Distinct error classes
    • Errors with this categorization will be grouped separately from other errors of its class. This can be useful when Airbrake groups errors too aggressively.

  • Strict error classes
    • Errors with this categorization will be grouped together using the whole backtrace. This can be useful when an exception is always raised from the same place.
